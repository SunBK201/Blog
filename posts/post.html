VLA is an acronym of variable-length array, which is an array (actual array, not just block of memory which can act like one) that have size (at least one dimension) determined during runtime (instead of at compile time).

It generates much more code, and much slower code (and more fragile code), than just using a fixed key size would have done ~ Linus Torvalds

当我在写Surge的规则配置文件时，我发现我需要通过正则表达式使用policy-regex-filter进行过滤， 具体的需求是：排除包含指定字符串的节点，过滤获得没有包含指定字符串的节点。 

      #include <stdio.h>
      
      int main(void) {
        int n;
        scanf("%d", &n);
        long double arr[n];
        printf("%Lf", arr[0]);
        return 0;
      }

作为一个软件开发者，我最高兴的是写出能被使用的软件。如果你的软件最终被锁在一个二进制的可执行文件中，必须购买、授权、运输、下载、安装、维护和升级，那么所有这些工艺的意义何在？由于程序员和用户之间存在着所有这些古老的、传统的障碍，软件业能够存在下去真是个奇迹。但在web应用的勇敢新世界里，这些限制都消失了。没有任何界限。软件可以无处不在。

As you could have guessed by the quote at the beginning, project which used to rely on VLA quite extensively is nothing else than Linux kernel. Maintainers spent a lot of effort to get rid of all VLA and as of version 4.20 (year 2018) it's completely VLA-free.
